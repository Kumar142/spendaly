❯ Why it is following only other environmnet? Why not env which I have          
activated?                                                                      

⏺ The venv exists but Flask was never installed into it. Here's what happened:

  1. You created the venv (python3 -m venv env) ✓
  2. But when you ran pip install -r requirements.txt, pip resolved to conda's 
  pip, not the venv's pip — so it installed into conda's environment, not your
  venv.
  3. The venv's Python has no packages in it.

  Fix — install directly into the venv:

  env/bin/pip install -r requirements.txt
  env/bin/python3 app.py

  To verify it worked:
  env/bin/pip list
  This should show only a small set of packages (just what's in
  requirements.txt), not the long conda list.

  Root cause: source env/bin/activate isn't overriding conda's PATH properly in
  your shell. Using env/bin/pip and env/bin/python3 directly bypasses this
  entirely.
